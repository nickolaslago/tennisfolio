Why Tennisfolio

Boring, beautiful, and it just works.

Everything you need to record and understand your game — nothing you don't.

Local-first

All your data in a single SQLite file, or your own Postgres container. Fully offline, and you own the backups.

One-screen match entry

Capture a whole match, sets and all, from a single form. Type 6-4 3-6 10-7 and the score parses itself.

Rich stats dashboard

Win-rate by surface, opponent and club; current and longest streaks; tiebreak and deciding-set records.

No account, no lock-in

Nothing to sign up for. A plain, documented schema and one-click export to CSV/JSON — your data is never trapped.

Self-host with Docker

One docker compose up runs the whole stack. Deploy it on TrueNAS, a homelab, or any box you control.

Open source

Public repo, AGPL-3.0 licensed. Read the code, file an issue, send a pull request — the roadmap is out in the open.

Install

Run it on your own hardware.

Tennisfolio is self-hosted. Your matches never leave the machine you run it on.

With Docker Image coming soon

A published container image is on the way. Until then, clone the repo and bring up the full stack — Postgres, API and web — with Docker Compose. 

# clone and start the whole stack
git clone https://github.com/nickolaslago/tennisfolio.git
cd tennisfolio
docker compose up -d

Then open:

  • Web app — http://localhost:3000
  • API docs — http://localhost:8000/docs

Once the image is published, a single docker run command will replace the clone step — this section will be updated then.

On TrueNAS SCALE

Perfect for a homelab: keep the database on your NAS and reach the app from any device on your network.

  1. Apps → Discover → Custom App (or Install via YAML) in the TrueNAS SCALE web UI.
  2. Point it at this repo's docker-compose.yml, or add the image once it's published.
  3. Map a host path for the Postgres volume so your data persists across updates.
  4. Set the web and API ports, deploy, and open the app from your browser.

A step-by-step TrueNAS guide lands alongside the published Docker image.

Prefer to run from source? See the Getting started guide in the README.
